# Отчёт по лабораторной работе №1
## Сравнение методов решения СЛАУ: Метод Гаусса и LU-разложение

### 1. Постановка задачи
Целью работы является реализация и сравнительный анализ алгоритмов решения систем линейных алгебраических уравнений (СЛАУ). 

**Необходимо реализовать:**
1. Метод Гаусса без выбора ведущего элемента.
2. Метод Гаусса с выбором главного элемента по столбцу.
3. LU-разложение матрицы без перестановок.
4. Прямую и обратную подстановки для использования с LU-разложением.

**Требуется провести эксперименты:**
- Сравнение времени работы алгоритмов для разных размеров матриц.
- Демонстрация преимущества LU-разложения при решении систем с одинаковой матрицей, но разными правыми частями.
- Оценка точности методов на плохо обусловленных матрицах (матрица Гильберта).

---

### 2. Описание реализованных алгоритмов

#### 2.1 Метод Гаусса
Алгоритм приводит расширенную матрицу системы $[A|b]$ к верхнетреугольному виду методом исключения переменных, после чего решение находится обратной подстановкой.

- **Без выбора ведущего элемента:** На шаге $k$ ведущим элементом считается $A_{kk}$. Прост в реализации, но может быть численно неустойчив при малых значениях $|A_{kk}|$.
- **С выбором главного элемента по столбцу:** Перед исключением на шаге $k$ строки переставляются так, чтобы модуль ведущего элемента $|A_{kk}|$ был максимальным в столбце $k$. Повышает устойчивость за счёт снижения накопления ошибок округления.

#### 2.2 LU-разложение
Матрица $A$ представляется в виде произведения $A = LU$, где:
- $L$ — нижняя треугольная матрица с единицами на диагонали,
- $U$ — верхняя треугольная матрица.

Решение системы $Ax = b$ сводится к двум этапам:
1. **Прямая подстановка:** решение $Ly = b$ относительно $y$.
2. **Обратная подстановка:** решение $Ux = y$ относительно $x$.

**Преимущество:** разложение $A = LU$ выполняется один раз, что делает метод эффективным при решении множества систем с одной матрицей $A$ и разными векторами $b$.

### 3. Структура программы

Программа написана на **C++ (стандарт C++11)**. Основные компоненты:

| Функция / Модуль | Назначение |
|-----------------|-----------|
| `gaussianEliminationNoPivot()` | Метод Гаусса без выбора ведущего |
| `gaussianEliminationPivot()` | Метод Гаусса с выбором по столбцу |
| `luDecomposition()` | LU-разложение матрицы (без перестановок) |
| `forwardSubstitution()` | Прямая подстановка ($Ly = b$) |
| `backwardSubstitution()` | Обратная подстановка ($Ux = y$) |
| `generateRandomMatrix()` | Генерация случайной матрицы с фиксированным seed |
| `generateHilbertMatrix()` | Генерация матрицы Гильберта $H_{ij} = 1/(i+j-1)$ |
| `computeRelativeError()` | Вычисление $\|\tilde{x} - x\| / \|x\|$ |
| `computeResidual()` | Вычисление невязки $\|A\tilde{x} - b\|$ |
| `experiment4_1/2/3()` | Реализация трёх серий экспериментов |
| `saveCSV()` | Сохранение результатов в CSV-файлы |

**Замер времени:** используется `std::chrono::high_resolution_clock` для высокоточного измерения.

### 4. Результаты экспериментов

#### 4.1 Сравнение времени решения одной системы
*Размеры матриц: $n = 100, 200, 500, 1000$.*

| n | Gauss (no pivot), мс | Gauss (pivot), мс | LU (total), мс | LU (decomp), мс | LU (solve), мс |
|---|---------------------|-------------------|----------------|-----------------|----------------|
| 100 | 2.838 | 2.933 | 3.106 | 3.044 | 0.063 |
| 200 | 22.021 | 31.315 | 30.319 | 29.928 | 0.391 |
| 500 | 363.661 | 371.928 | 385.802 | 383.081 | 2.721 |
| 1000 | 3124.209 | 3116.886 | 3415.530 | 3409.646 | 5.883 |

**Наблюдения:**
- Время работы всех методов растёт пропорционально $O(n^3)$, что соответствует теоретической сложности.
- Метод Гаусса с выбором ведущего элемента работает сопоставимо с методом без выбора (иногда даже быстрее из-за особенностей кэширования и ветвления процессора).
- Время решения треугольных систем в LU-методе ($\approx$ 0.06–5.9 мс) значительно меньше времени разложения, что подтверждает эффективность при многократном использовании.

#### 4.2 Экономия времени при множественных правых частях
*Размер матрицы: $n = 500$. Количество правых частей $k = 1, 10, 100$.*

| k | Gauss (total), мс | LU (total), мс | Выигрыш LU |
|---|------------------|----------------|-----------|
| 1 | 350.23 | 376.54 | −7.5% |
| 10 | 3578.96 | 381.87 | **9.4×** |
| 100 | 41383.89 | 555.40 | **74.5×** |

**Наблюдения:**
- При $k = 1$ LU-метод немного медленнее из-за накладных расходов на разложение.
- При $k = 10$ выигрыш составляет почти 10 раз.
- При $k = 100$ LU-метод быстрее **в 74 раза**, так как дорогостоящее разложение выполняется один раз, а решение треугольных систем имеет сложность $O(n^2)$.

#### 4.3 Проверка точности на матрице Гильберта
*Матрица Гильберта $H_{ij} = 1/(i+j-1)$ является классическим примером плохо обусловленной матрицы.*

| n | Метод | Относительная погрешность | Невязка |
|---|-------|--------------------------|---------|
| 5 | Gauss (no pivot) | 0.000000 | 0.000000 |
| 5 | Gauss (pivot) | 0.000000 | 0.000000 |
| 5 | LU | 0.000000 | 0.000000 |
| 10 | Gauss (no pivot) | 2.24·10⁻⁴ | 0.000000 |
| 10 | Gauss (pivot) | 1.63·10⁻⁴ | 0.000000 |
| 10 | LU | 3.07·10⁻⁴ | 0.000000 |
| 15 | Gauss (no pivot) | 9.48·10⁻¹ | 0.000000 |
| 15 | Gauss (pivot) | 8.52·10⁰ | 0.000000 |
| 15 | LU | 2.82·10⁰ | 0.000000 |

**Наблюдения:**
- При $n = 5$ все методы дают точное решение (в пределах машинной точности).
- При $n = 10$ погрешность растёт, но остаётся приемлемой ($\sim 10^{-4}$).
- При $n = 15$ матрица становится крайне плохо обусловленной ($\text{cond}(H_{15}) \approx 10^{17}$), и все методы демонстрируют значительную потерю точности.
- Неожиданно, метод с выбором ведущего элемента показал **наибольшую погрешность** (8.52). Это объясняется тем, что для матриц Гильберта выбор ведущего элемента не всегда улучшает устойчивость из-за специфики структуры матрицы и накопления ошибок округления при перестановках.
- Невязка во всех случаях близка к нулю, что говорит о том, что найденное решение удовлетворяет системе $A\tilde{x} \approx b$, но может сильно отличаться от точного $x$ из-за плохой обусловленности.

### 5. Выводы

1. **Эффективность при множественных правых частях:**  
   LU-разложение демонстрирует существенное преимущество при решении систем с одной матрицей и разными правыми частями. При $k = 100$ выигрыш во времени составляет **74 раза**, что полностью подтверждает теоретические ожидания.

2. **Влияние выбора ведущего элемента:**  
   - Для случайных матриц выбор ведущего элемента не даёт значительного выигрыша во времени, но повышает численную устойчивость.
   - Для плохо обусловленных матриц (Гильберта) выбор ведущего элемента не гарантирует улучшения точности и в некоторых случаях может даже ухудшить результат из-за специфики структуры матрицы.

3. **Точность на плохо обусловленных системах:**  
   Все прямые методы (Гаусс, LU) теряют точность при росте числа обусловленности матрицы. Для таких задач рекомендуется использовать итерационные методы уточнения или алгоритмы с повышенной точностью (например, с использованием арифметики произвольной точности).

4. **Практическая рекомендация:**  
   - Для общих задач использовать **метод Гаусса с выбором ведущего элемента** как баланс между скоростью и устойчивостью.
   - При решении множества систем с одной матрицей — **предварительно выполнить LU-разложение**.
   - Для плохо обусловленных систем — применять специализированные методы (регуляризация, итерационное уточнение).
