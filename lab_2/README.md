## 🔧 Технические детали реализации

### Архитектура класса `SingleLayerPerceptron`

```python
class SingleLayerPerceptron:
    ├── __init__(w_init, b_init, seed)   # инициализация параметров
    ├── _init_params(n_features)          # малые случайные веса ~N(0, 0.01)
    ├── sigmoid(z) [static]               # σ(z) = 1/(1+e⁻ᶻ) с защитой от overflow
    ├── forward(X)                        # ŷ = σ(X·w + b) — прямой проход
    ├── compute_loss(...)                 # BCE или Hinge + L2-регуляризация
    ├── fit(X_train, y_train, X_val, ...) # обучение с мини-батчами и моментом
    ├── predict(X)                        # бинарные метки (порог 0.5)
    └── predict_proba(X)                  # вероятности класса 1
```

### Ключевые формулы

| Компонент | Формула | Код |
|-----------|---------|-----|
| **Прямой проход** | $z = w^T x + b$, $\hat{y} = \sigma(z)$ | `X @ self.w + self.b` |
| **BCE Loss** | $L = -\frac{1}{m}\sum[y\log\hat{y} + (1-y)\log(1-\hat{y})]$ | `compute_loss()` |
| **Градиенты** | $\frac{\partial L}{\partial w} = \frac{1}{m}X^T(\hat{y}-y)$ | `dw = (1/m) * (X_b.T @ (y_pred - y_b))` |
| **Обновление** | $w \leftarrow w - \eta \cdot \frac{\partial L}{\partial w}$ | `self.w -= lr * dw` |
| **L2-градиент** | $\frac{\partial L_{L2}}{\partial w} += \frac{\lambda}{m} w$ | `if l2_lambda > 0: dw += ...` |

## 🔍 Ответы на контрольные вопросы

**1. Почему стандартизация считается только по train?**  
Чтобы избежать *data leakage*: если посчитать µ и σ по всему датасету, модель косвенно «увидит» статистику тестовых данных, и метрики будут завышены.

**2. Почему Recall = 0.933, а Precision = 0.854?**  
Модель «щедро» предсказывает класс 1 (низкий порог уверенности), поэтому находит почти все реальные положительные примеры (высокий Recall), но допускает больше ложных срабатываний (ниже Precision).

**3. Почему перцептрон не решает XOR?**  
XOR не является линейно разделимой задачей: нельзя провести одну прямую, разделяющую классы. Однослойный перцептрон строит только линейную границу $w^T x + b = 0$.

**4. Зачем нужен Momentum?**  
Он добавляет «инерцию» обновлению весов, помогая:  
- Ускорить сходимость в узких оврагах функции потерь  
- Сгладить шум от мини-батчей  
- Избежать мелких локальных минимумов
